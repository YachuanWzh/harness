# Ralph Auto-Trigger + Relocate to .claude/superharness

**Goal:** Make ralph task-tracking files (`.current-task`, `task.json`, `trace.jsonl`,
`.ralph-state.json`) (1) appear **automatically** when a `/superharness:go` task starts â€”
no agent-run bootstrap required â€” and (2) live under `.claude/superharness/ralph/`
instead of `superharness/ralph/`.

**Why:** Today the files only appear if the agent remembers to run the Phase-1
bootstrap commands; nothing enforces it, so a whole complex run can produce zero
tracking. And the path was the project root, not under `.claude/superharness`.

**Key facts established:**
- Claude Code `UserPromptSubmit` hook receives the **raw typed prompt** including the
  slash command (verified against code.claude.com/docs/en/hooks). So the hook can
  detect `/superharness:go <goal>`.
- `lib/install.ps1` copies `template/* -> .claude/superharness/` with `-Force`
  (overwrite, not delete-then-copy), so a sibling `.claude/superharness/ralph/`
  runtime dir survives re-installs.
- In target projects `.claude/superharness/` is committed (plugin travels with repo),
  so the installer must gitignore **`.claude/superharness/ralph/`** (runtime only).
- Source of truth is `template/plugins/superharness/...`; `.claude/superharness/` here
  is gitignored dogfood, regenerated by the installer.

**Tech:** Windows PowerShell 5.1, zero-dep. Tests: `tests/run-tests.ps1`
(`Assert-True` harness). Run: `powershell -NoProfile -ExecutionPolicy Bypass -File tests/run-tests.ps1`.

## Task 2 â€” `Get-RalphGoInvocation` pure parser (ralph-lib)

Detect a go invocation in a prompt and derive a dated kebab slug + goal.

- Contract: `Get-RalphGoInvocation -Prompt <text> [-Now <DateTime>]` â†’
  `$null` if prompt is not a `/superharness:go` (or `superharness:go`) invocation;
  else `[pscustomobject]@{ Goal=<trimmed goal>; Slug='YYYY-MM-DD-<kebab|task>' }`.
  Kebab = ASCII `[a-z0-9]+` tokens of the goal joined by `-`, capped ~6 tokens; when
  no ASCII tokens (e.g. Chinese goal) fall back to `task-HHmmss` (from `-Now`).
- RED: new test group asserting match/no-match, slug shape, goal extraction, CJK fallback.
